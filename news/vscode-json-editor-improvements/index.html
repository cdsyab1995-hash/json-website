VS Code 1.90 Adds Advanced JSON Editing Features

Published: 2026-05-04 · 1 min read · 274 words

VS Code 1.90: Enhanced JSON Support

Visual Studio Code 1.90, released in April 2026, includes significant improvements to JSON editing capabilities. The update adds inline validation errors, automatic schema-based completions, and JSON refactoring commands that streamline common editing tasks.

The inline validation feature highlights JSON Schema violations directly in the editor, showing error messages as you type. This immediate feedback helps developers catch errors early, before running code or tests. The validation is powered by the file's JSON Schema, if available, or falls back to basic syntax validation.

The new refactoring commands support common JSON transformations: extracting properties to nested objects, inlining nested objects, moving properties between objects, and converting between array and object representations. These refactorings work with JSON Schema awareness, ensuring that refactored JSON remains valid against the schema.

Improved JSON Schema Support

VS Code 1.90 improves its handling of JSON Schema, including better support for schemas with references, improved schema resolution for large files, and enhanced preview of schema validation results. The editor now caches schema information more aggressively, reducing validation latency for large JSON files.

The release also adds support for JSON Schema draft 2020-12, ensuring compatibility with the latest specification. Schemas written for earlier drafts continue to work without modification, maintaining backward compatibility for existing projects.

Developer Experience Improvements

Beyond the technical improvements, VS Code 1.90 includes quality-of-life features for JSON developers. The JSON outline view has been improved to handle large files better and provide more navigation options. The Go to Symbol feature (Ctrl+Shift+O) now works across the entire JSON document, not just the root level.

These improvements continue VS Code's tradition of providing excellent out-of-the-box JSON support, making it a strong choice for developers who work extensively with JSON data.
