VS Code 1.114: AI Takes Over JSON Editing with Schema Inference and Inline Fixes

Published: 2026-04-24 · 4-5 min read

Visual Studio Code 1.114 represents the most significant upgrade to JSON editing in the editor's history. The April 2026 release ships three AI-native JSON features that fundamentally change how developers work with structured data: real-time schema inference, Copilot-powered inline error explanations, and a new JSON Query Assistant that understands JSONPath and jq syntax.

Feature 1: Real-Time JSON Schema Inference

Previously, getting JSON validation in VS Code required manually pointing a file to a JSON Schema via settings.json or schema store. In 1.114, VS Code automatically infers a schema from the document structure as you type and shows validation errors without any configuration.

The inference engine uses a combination of static analysis and an on-device AI model to detect:

  • Common API response patterns (REST, GraphQL, tRPC)
  • Well-known config file formats (package.json, tsconfig, GitHub Actions)
  • Custom schemas inferred from sibling files in the same workspace
  • Types derived from TypeScript interfaces in the project
// You paste this JSON in a new file — no schema configured:
{
    "user": {
        "id": 42,
        "name": "Alice",
        "role": "admin",
        "createdAt": "2026-01-15"
    },
    "permissions": ["read", "write", "delete"],
    "quota": {
        "used": 850,
        "limit": 1000,
        "unit": "MB"
    }
}

// VS Code 1.114 automatically:
// 1. Infers the schema (id: integer, name: string, role: enum, etc.)
// 2. Shows hover types for each field
// 3. Warns if you add "role": 123 (type mismatch)
// 4. Suggests autocompletions based on inferred enum values

Feature 2: AI-Powered Inline Error Explanations

When JSON validation fails, VS Code 1.114 replaces the cryptic schema error messages with plain-English explanations generated by GitHub Copilot. Hovering over a red underline now shows a panel with:

  • What went wrong: A human-readable description of the validation failure
  • Why it matters: Context about why the schema requires this format
  • How to fix it: One-click suggested corrections
  • Reference: Links to the relevant JSON Schema specification section

Before this feature, a typical JSON Schema validation error looked like:

// Old VS Code error message:
"Value is not accepted. Valid values: 1, 2, 3."

// VS Code 1.114 AI explanation:
"The 'status' field expects a numeric status code (1, 2, or 3),
but you provided the string "active". This API uses numeric
status codes where: 1=active, 2=pending, 3=disabled.
Try: "status": 1"

Feature 3: JSON Query Assistant

The most ambitious new feature is the JSON Query Assistant, accessible via Ctrl+Shift+J (or Cmd+Shift+J on Mac). It opens a side panel where you can query the current JSON document using natural language, JSONPath, or jq syntax — with AI translation between all three.

// Natural language query:
"Show me all users with role admin and quota over 80% used"

// VS Code translates to JSONPath:
$.users[?(@.role == "admin" && @.quota.used / @.quota.limit > 0.8)]

// And to jq:
.users[] | select(.role == "admin" and (.quota.used / .quota.limit) > 0.8)

// Results are shown inline with syntax highlighting

Python Extension: Experimental Rust Indexer

Alongside JSON improvements, VS Code 1.114 ships an experimental Rust-based indexer for the Python extension. For large Python projects that also use JSON configuration files, this indexer runs in parallel with the TypeScript language server, providing:

  • 10x faster project indexing on large codebases
  • Parallel JSON schema validation alongside Python type checking
  • Reduced memory contention between language servers

Key Takeaways

  • VS Code 1.114's AI schema inference means zero-config JSON validation for most use cases
  • Human-readable error explanations dramatically reduce JSON debugging time
  • The JSON Query Assistant bridges the gap between developers unfamiliar with JSONPath/jq syntax
  • For complex JSON validation and formatting tasks, AI JSON's online tools remain a fast complement to in-editor tooling
