Bun 1.2 Sets New Record for JSON HTTP Server Performance

Published: 2026-05-04 · 2 min read · 314 words

Bun's New JSON HTTP Server Benchmark Results

Bun 1.2, the JavaScript runtime known for its speed, has released benchmark results for its new JSON HTTP server that show dramatic performance improvements over Node.js. In tests measuring requests per second for JSON API endpoints, Bun 1.2 achieves approximately 2.5x the throughput of Node.js 22.

The benchmark tests realistic JSON API scenarios: parsing incoming JSON requests, processing the data, generating JSON responses, and sending them back. The tests use production-representative payloads of moderate size (1-10KB per request) and measure sustained throughput over extended periods.

The performance advantage comes from several factors: Bun's JavaScript engine optimizations, native HTTP handling that bypasses some Node.js abstractions, and streaming JSON parsing that reduces memory overhead. For applications that are JSON-heavy, these advantages translate directly to lower infrastructure costs and better user experiences.

Implications for Production Deployments

The benchmark results have implications for organizations evaluating JavaScript runtimes for production workloads. For applications where JSON processing is a significant portion of total processing time, Bun's performance advantages can reduce infrastructure requirements.

However, the benchmark results should be evaluated in context. Real-world applications have varying profiles: some are JSON-heavy, others are database-heavy, and others spend significant time in custom business logic. Organizations should benchmark their actual applications rather than relying solely on synthetic benchmarks.

Ecosystem compatibility also matters. Bun has made significant progress on Node.js compatibility, but some packages and native modules may still have issues. Organizations considering Bun for production should carefully evaluate their dependency trees and test thoroughly.

Future Performance Improvements

The Bun team has indicated that further performance improvements are in development. Upcoming versions will include additional optimizations for JSON streaming and HTTP/2 support, which should further extend Bun's performance lead.

The focus on JSON performance reflects Bun's positioning as an ideal runtime for modern web applications that rely heavily on JSON for data exchange. As JSON remains the dominant format for web APIs, runtimes optimized for JSON processing provide tangible benefits for developers.
