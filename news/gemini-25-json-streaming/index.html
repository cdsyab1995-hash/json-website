Gemini 2.5 Pro API Ships Native JSON Streaming with Schema Conformance

Published: 2026-04-25 | Reading time: 6 minutes

Google has released a major update to the Gemini 2.5 Pro API that introduces native JSON streaming with guaranteed schema conformance. Previously, developers had to collect the full model response before parsing JSON. With this release, structured output tokens arrive in real time — fully validated against the provided JSON Schema as they stream.

What's New in Gemini 2.5 Pro Streaming JSON

1. Streaming Structured Outputs

The updated API adds a response_schema parameter to the streaming endpoint. When provided, Gemini uses constrained decoding to guarantee every token it emits forms a valid partial JSON object that will eventually satisfy the schema. Developers no longer need to buffer responses and parse after the fact.

{
  "model": "gemini-2.5-pro",
  "stream": true,
  "response_mime_type": "application/json",
  "response_schema": {
    "type": "object",
    "properties": {
      "title": { "type": "string" },
      "tags": { "type": "array", "items": { "type": "string" } },
      "confidence": { "type": "number" }
    },
    "required": ["title", "tags", "confidence"]
  }
}

2. Live Token Budget Reporting

A new token_budget field in the streaming response metadata allows developers to track token consumption in real time. This is particularly useful for cost-sensitive applications that need to dynamically truncate responses or switch to a smaller model mid-stream.

3. Multi-Turn JSON Conversation Memory

Gemini 2.5 Pro now supports multi-turn conversation sessions where the JSON schema is enforced across the entire conversation. Each model turn in the session produces JSON that references the same schema, enabling stateful structured data extraction over long interactions — ideal for data collection bots and interview agents.

4. Partial Object Streaming Callbacks

The new on_partial_object callback fires whenever a complete nested object is received within the stream. For example, if your schema has an array of items, each item triggers the callback as soon as it's fully received — before the overall response is complete. This dramatically reduces first-meaningful-data latency for list-based responses.

Performance Benchmarks

Google's internal benchmarks compare streaming JSON parsing with and without the new native mode:

Scenario Old (Buffer + Parse) New (Native Stream) Improvement
Time to first object property ~4,200ms ~180ms 23x faster
Schema validation errors caught After full response At token level Instant feedback
Memory usage (100KB response) ~8MB buffer ~0.3MB stream 96% reduction

Real-World Use Cases

AI Data Extraction Pipelines

Teams building document intelligence pipelines can now stream-extract structured data from PDFs, emails, and web pages with real-time JSON output. The partial object callback means downstream systems can start processing extracted entities while the model is still reading the document.

LLM-Powered API Backends

When Gemini powers a REST API backend, streaming JSON eliminates the latency cost of buffering. Client applications receive the first data fields within milliseconds of the model starting generation, rather than waiting for the full response.

Agentic Tool Calls

MCP servers and AI agent frameworks benefit directly. Tool call arguments arrive as streaming JSON, allowing agents to begin acting on early parameters before the full tool call is received — enabling sub-100ms tool invocation latency in favorable conditions.

Migration Guide

For existing Gemini API users, migrating to streaming JSON requires two changes:

  1. Add "stream": true to the API request body
  2. Add "response_schema": { ... } with your existing schema definition
  3. Update your response handler to process chunks via the on_partial_object callback or standard SSE chunk parsing

The old buffered JSON mode remains available and is not deprecated. Both modes coexist in the same API version.

Availability

Native JSON streaming is available today in the Gemini 2.5 Pro API at stable tier. Gemini 2.5 Flash support is expected in the next monthly release. The feature is accessible via both the REST API and all official SDKs (Python, Node.js, Go, Java).
