Gemini 2.5 Introduces Native JSON Streaming: Parse AI Responses Progressively as Tokens Arrive

Published: 2026-04-29 · 5-6 min read

Google has released the Gemini 2.5 API with a headline feature that changes how developers consume structured AI outputs: native JSON streaming. Instead of waiting for the complete JSON response before parsing, the new streamStructured() method lets applications process and render JSON data progressively as tokens stream from the model — delivering the same latency improvements that text streaming brought to chat interfaces, now for structured data pipelines.

The Structured Output Latency Problem

Text streaming in LLM APIs is well-established. Chat interfaces have displayed tokens word-by-word for years. But when developers switched to structured outputs — asking the model to return JSON — they lost streaming entirely. JSON is only valid when complete; you can't parse {"name": "Alice", "age": because it's a syntax error. So structured output pipelines were forced back to a blocking "wait for the full response" model.

For long structured outputs — detailed extraction tasks, multi-field reports, large array responses — this meant waiting 5-15 seconds before any data was available. With Gemini 2.5's streaming approach, the first valid JSON objects from an array response are available in under a second.

How streamStructured() Works

The new API uses a streaming JSON parser on the client side that maintains a partial parse state as tokens arrive. Rather than waiting for the final closing brace, it emits complete JSON objects as soon as they're syntactically complete within the stream:

import { GoogleGenerativeAI } from '@google/generative-ai';

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
const model = genAI.getGenerativeModel({ model: 'gemini-2.5-pro' });

const stream = await model.streamStructured({
  prompt: 'Extract all product mentions from this review: ...',
  schema: {
    type: 'array',
    items: {
      type: 'object',
      properties: {
        name: { type: 'string' },
        sentiment: { type: 'string', enum: ['positive', 'negative', 'neutral'] },
        price_mentioned: { type: 'boolean' }
      },
      required: ['name', 'sentiment', 'price_mentioned']
    }
  }
});

// Items are emitted as they complete in the stream
for await (const item of stream) {
  console.log('Got item:', item); // Each product as it's parsed
  updateUI(item); // Can render incrementally
}

Partial Object Streaming

Beyond array item streaming, Gemini 2.5 also supports partial object streaming — receiving incomplete objects with the fields that have been generated so far. This is useful for showing progressive form fills or live-updating UI components as the model generates each field:

const stream = await model.streamStructured({
  prompt: 'Generate a detailed product description',
  schema: ProductSchema,
  mode: 'partial' // Enable partial object streaming
});

for await (const partial of stream) {
  // partial might be: { name: "Wireless Headphones" }
  // then: { name: "Wireless Headphones", price: 89.99 }
  // then: { name: "...", price: ..., description: "Premium quality..." }
  renderPartialProduct(partial);
}

Schema Validation at Stream Time

Each emitted item is validated against the provided schema before being yielded. If the model generates a malformed item (wrong type, missing required field), it is caught at stream time and an error event is emitted rather than a data item. This means downstream code never receives invalid JSON objects, and the pipeline can recover gracefully from individual item failures without abandoning the entire stream.

Real-World Performance

Google's internal benchmarks on a 50-item product extraction task (typical for e-commerce catalog enrichment) show:

  • Gemini 2.0 non-streaming: 8.2 seconds to first usable data
  • Gemini 2.5 with streamStructured: 0.8 seconds to first item, 8.4 seconds to all 50 items

The total processing time is essentially identical — the streaming overhead is minimal. But the first-usable-data latency drops by 10x, enabling UI patterns that were previously impractical with structured outputs.

Compatibility with JSON Tools

Since each streamed item is valid JSON, it's fully compatible with standard JSON processing tools. The streaming nature doesn't add any complexity to downstream tooling — each item is just a regular JSON object.

Availability

JSON streaming is available in the Gemini 2.5 Pro and Gemini 2.5 Flash tiers. It requires the Google AI SDK version 0.14.0 or higher. The feature is production-ready and not behind any experimental flags.
