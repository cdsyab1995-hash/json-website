JSONata 2.0 Ships AI-Powered Query Suggestions: Natural Language to JSONata Expressions

Published: 2026-04-29 · 5-6 min read

JSONata — the JSON query and transformation language popular in Node-RED, AWS Step Functions, and IBM App Connect — has released version 2.0 with a headline feature that makes the language significantly more accessible: an AI-powered query assistant that translates natural language descriptions into JSONata expressions. For developers who know what they want to do with JSON but struggle with JSONata's syntax, this changes the workflow substantially.

What Is JSONata?

JSONata is a lightweight query and transformation language for JSON data. It's particularly popular in integration platforms (Node-RED, MuleSoft, AWS Step Functions) where developers need to transform JSON from one shape to another. A simple JSONata expression can flatten nested arrays, filter by conditions, aggregate values, and reshape entire document structures — but the syntax has a steep learning curve for newcomers.

The AI Query Assistant

JSONata 2.0's AI assistant takes a natural language description of the desired transformation and the input JSON structure, then generates a working JSONata expression:

import { suggest } from 'jsonata/ai';

const inputJSON = {
  orders: [
    { id: 'o1', customer: 'Alice', items: [{ sku: 'A1', qty: 2, price: 10 }] },
    { id: 'o2', customer: 'Bob', items: [{ sku: 'B1', qty: 1, price: 25 }] }
  ]
};

const expression = await suggest(
  inputJSON,
  'Get the total value of each order (sum of qty * price for each item) with the order id and customer name'
);

console.log(expression);
// Output: orders.{ "id": id, "customer": customer, "total": $sum(items.(qty * price)) }

// Run it:
const result = await jsonata(expression).evaluate(inputJSON);
// [{ id: 'o1', customer: 'Alice', total: 20 }, { id: 'o2', customer: 'Bob', total: 25 }]

How the AI Was Trained

The JSONata team fine-tuned a small language model on a dataset of 50,000+ JSONata expression/description pairs, drawn from GitHub repositories, Stack Overflow questions, and the official documentation examples. The model runs locally in Node.js via ONNX Runtime — no API calls to external services, no data sent off-device. For production integration platforms that handle sensitive data, this privacy-preserving approach matters.

JSONata 2.0 Core Changes

Beyond the AI assistant, JSONata 2.0 includes several long-requested language improvements:

  • Async expressions: $async() function for calling external services within a transformation pipeline
  • Named lambdas: Functions can now be defined with names for better debugging and recursion
  • Streaming evaluation: Large input documents can be evaluated as a stream, avoiding loading the full JSON into memory
  • TypeScript-first SDK: The npm package now ships full TypeScript types for all APIs

Integration Ecosystem

JSONata 2.0 maintains backward compatibility with all 1.x expressions. AWS has announced JSONata 2.0 support in Step Functions will ship in Q3 2026. The Node-RED team plans to upgrade their default JSONata version in Node-RED 4.1.
