VS Code GitHub Copilot Now Auto-Generates JSON Schemas from TypeScript Interfaces

Published: 2026-04-29 · 5-6 min read

GitHub Copilot's April 2026 update to VS Code introduces one of the most requested developer workflows: automatic JSON Schema generation directly from TypeScript interfaces. The feature closes the longstanding gap between TypeScript type safety and JSON validation at runtime.

TypeScript to JSON Schema in One Click

Select any TypeScript interface in VS Code, right-click, and choose 'Generate JSON Schema'. Copilot analyzes the interface, resolves referenced types, and produces a valid JSON Schema Draft 2020-12 document: 

// TypeScript Interface
interface UserConfig {
  id: string;
  name: string;
  role: 'admin' | 'user' | 'viewer';
  settings: {
    theme: 'light' | 'dark';
    notifications: boolean;
  };
  tags?: string[];
}

// Generated JSON Schema
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "id": { "type": "string" },
    "role": { "enum": ["admin", "user", "viewer"] },
    "settings": {
      "type": "object",
      "properties": {
        "theme": { "enum": ["light", "dark"] },
        "notifications": { "type": "boolean" }
      }
    }
  },
  "required": ["id", "name", "role", "settings"]
}

Bidirectional Schema Sync

The feature supports bidirectional sync — changes to either the TypeScript interface or the JSON Schema are reflected in both files. This eliminates the drift that typically occurs when schemas are manually maintained.

AI-Powered Validation Fixes

When Copilot detects that a JSON file doesn't match its associated schema, it now suggests specific fixes inline. Rather than just flagging an error, it explains what's wrong and offers a one-click correction — including type coercions, missing required fields, and enum mismatches.

Impact on API Development

Teams at several major companies have reported 60% reduction in JSON validation bugs caught only at runtime, faster onboarding since schemas document API contracts automatically, and elimination of manual schema maintenance overhead estimated at 2-3 hours per week per developer.
