OpenAI Agents SDK Ships Strict JSON Mode: Guaranteed Schema Conformance

Published: 2026-04-23 · 5-6 min read

OpenAI has released strict JSON mode in the Agents SDK, solving one of the most persistent pain points for developers building production AI pipelines: unpredictable structured outputs. With strict mode enabled, LLM responses are guaranteed to conform to your JSON Schema — no extra fields, no missing keys, no type mismatches.

What Is Strict JSON Mode?

When you define a JSON Schema for a tool call or function output, the Agents SDK now enforces schema conformance at the API level. The model's tokenizer is constrained to only produce valid JSON that matches your schema — a technique known as constrained decoding.

from openai_agents import Agent, tool

@tool(
    name="get_weather",
    strict=True,  # Enable strict JSON mode
    output_schema={
        "type": "object",
        "properties": {
            "temperature": {"type": "number"},
            "condition": {"type": "string", "enum": ["sunny", "cloudy", "rainy"]},
            "humidity": {"type": "integer", "minimum": 0, "maximum": 100}
        },
        "required": ["temperature", "condition", "humidity"],
        "additionalProperties": false
    }
)
def get_weather(location: str):
    # LLM output is guaranteed to match the schema above
    ...

Why This Matters

Before strict mode, developers had to handle JSON parsing errors, retry logic, and validation in application code. The failure modes were unpredictable:

  • Models would occasionally add fields not in the schema
  • String fields might return numbers, breaking downstream parsers
  • Required fields could be omitted under edge-case prompts
  • Nested objects were particularly prone to structural drift

With strict mode, these failure modes are eliminated at the SDK level. The API guarantees schema conformance or returns an error, never a malformed partial response.

How Constrained Decoding Works

Under the hood, strict JSON mode uses a grammar-constrained sampling approach:

  1. Your JSON Schema is compiled into a context-free grammar
  2. At each decoding step, only tokens that produce valid grammar continuations are allowed
  3. The model selects the highest-probability valid token at every step
  4. Output is 100% guaranteed to be parseable and schema-valid
# Before strict mode - defensive coding required
try:
    data = json.loads(response.content)
    # Manual validation
    assert "temperature" in data
    assert isinstance(data["temperature"], (int, float))
    assert data["condition"] in ["sunny", "cloudy", "rainy"]
except (json.JSONDecodeError, AssertionError, KeyError) as e:
    # Retry logic, fallback handling...
    handle_parse_error(e)

# With strict mode - just use the data
data = response.structured_output  # Already validated
print(f"Temp: {data['temperature']}°C, {data['condition']}")

Performance Trade-offs

Constrained decoding has a small performance cost. In OpenAI's benchmarks:

  • Latency: +8-15ms per request (schema compilation is cached)
  • Throughput: ~5% reduction vs. unconstrained generation
  • Accuracy: 100% schema conformance vs. 94.7% in standard mode
  • Cost: No additional token cost — schema validation is compute-side

Integration with Zod and TypeScript

For TypeScript developers, the SDK includes a Zod bridge that auto-generates JSON Schemas from your Zod types:

import { z } from 'zod';
import { zodToJsonSchema } from 'zod-to-json-schema';
import { Agent, tool } from '@openai/agents';

const WeatherSchema = z.object({
    temperature: z.number(),
    condition: z.enum(['sunny', 'cloudy', 'rainy']),
    humidity: z.number().int().min(0).max(100)
});

const weatherTool = tool({
    name: 'get_weather',
    strict: true,
    outputSchema: zodToJsonSchema(WeatherSchema),
    execute: async ({ location }) => {
        // TypeScript types are inferred from Zod schema
        const result: z.infer<typeof WeatherSchema> = await fetchWeather(location);
        return result;
    }
});

Key Takeaways

  • Strict JSON mode eliminates schema validation code from your application layer
  • Constrained decoding guarantees conformance — not just high probability
  • Small performance cost (8-15ms) is worth the reliability gain for production systems
  • Works seamlessly with Zod for TypeScript developers
