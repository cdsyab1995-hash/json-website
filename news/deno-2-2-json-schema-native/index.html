Deno 2.2 Ships Native JSON Schema Validation: No More Third-Party Dependencies

Published: 2026-04-25 · 5-6 min read

Deno 2.2 has landed, and its headline feature is something the JavaScript ecosystem has needed for years: native JSON Schema validation built directly into the runtime. No Ajv, no zod-to-json-schema, no third-party deps — just Deno.validate(), ready out of the box.

Why Native JSON Schema Validation Matters

JSON Schema is the de-facto standard for describing and validating JSON data structures. It powers OpenAPI specs, configuration files, LLM tool schemas, and countless API contracts. Yet every JavaScript runtime until now has required developers to import a third-party library — adding hundreds of kilobytes to bundle sizes and introducing dependency management overhead.

Deno's approach is different: the team embedded a high-performance, Rust-based JSON Schema validator directly into the runtime core. The implementation targets the JSON Schema Draft 2020-12 specification — the latest and most feature-complete version.

The New API: Deno.validate()

The new API is intentionally minimal. Here's the core interface:

// Basic usage
const schema = {
    type: "object",
    properties: {
        name: { type: "string", minLength: 1 },
        age: { type: "integer", minimum: 0, maximum: 150 },
        email: { type: "string", format: "email" }
    },
    required: ["name", "email"],
    additionalProperties: false
};

const result = Deno.validate(schema, {
    name: "Alice",
    age: 30,
    email: "alice@example.com"
});

console.log(result.valid);   // true
console.log(result.errors);  // []

// Validation failure
const bad = Deno.validate(schema, {
    name: "",           // minLength violation
    age: -5,            // minimum violation
    extra: "field"      // additionalProperties violation
});

console.log(bad.valid);   // false
console.log(bad.errors);  // Array of ValidationError objects

Compile-Once, Validate-Many Pattern

For high-throughput use cases — think API servers validating thousands of requests per second — Deno 2.2 provides a compiled schema interface:

// Compile the schema once at startup
const validator = Deno.compileSchema(schema);

// Use the compiled validator for each request
Deno.serve(async (req) => {
    const body = await req.json();
    const result = validator.validate(body);

    if (!result.valid) {
        return Response.json(
            { error: "Invalid request", details: result.errors },
            { status: 400 }
        );
    }

    // Process valid request...
    return Response.json({ ok: true });
});

Full Draft 2020-12 Support

Unlike many validators that only partially implement JSON Schema, Deno's native validator fully supports Draft 2020-12 features that developers actually need in 2026:

  • Composition keywords: if/then/else, dependentSchemas, dependentRequired
  • String annotations: format validation (email, uri, date-time, ipv4, ipv6)
  • Array control: prefixItems, unevaluatedItems, contains
  • Object control: unevaluatedProperties, patternProperties
  • References: $ref, $defs, recursive schemas
  • Vocabulary: Custom vocabulary support for specialized domains
