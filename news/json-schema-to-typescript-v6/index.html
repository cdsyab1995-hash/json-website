json-schema-to-typescript v6: Draft 2020-12 Support, Discriminated Unions, and Cleaner Types

Published: 2026-04-29 · 4-5 min read

The most widely used JSON Schema to TypeScript code generator has shipped version 6, and it's a significant quality leap. json-schema-to-typescript v6 adds full JSON Schema Draft 2020-12 support, generates proper TypeScript discriminated unions from oneOf schemas with a discriminator field, and produces significantly cleaner type definitions for complex nested schemas. For teams maintaining TypeScript types derived from OpenAPI specs or JSON Schema libraries, v6 reduces manual type cleanup substantially.

Draft 2020-12 Support

Previous versions of the library supported Draft 4 and Draft 7, but Draft 2020-12 schemas (increasingly common in new tooling) were handled inconsistently. V6 adds full support for the 2020-12 vocabulary, including:

  • prefixItems for typed tuple schemas (replaces the Draft 7 items array syntax)
  • unevaluatedProperties and unevaluatedItems constraints
  • $dynamicRef and $dynamicAnchor for recursive schemas
  • The updated $vocabulary system for schema composition

Discriminated Unions

The most impactful improvement is the generation of proper TypeScript discriminated unions from oneOf schemas that include a discriminator field. Previously, these generated wide union types that TypeScript couldn't narrow automatically:

// Input JSON Schema
{
  "oneOf": [
    { "properties": { "type": { "const": "cat" }, "indoor": { "type": "boolean" } }, "required": ["type", "indoor"] },
    { "properties": { "type": { "const": "dog" }, "breed": { "type": "string" } }, "required": ["type", "breed"] }
  ],
  "discriminator": { "propertyName": "type" }
}

// v5 output (not narrowable):
type Animal = { type: "cat"; indoor: boolean } | { type: "dog"; breed: string };

// v6 output (proper discriminated union with TypeScript tag):
type Animal =
  | { type: "cat"; indoor: boolean }
  | { type: "dog"; breed: string };

// Now TypeScript can narrow automatically:
if (animal.type === "cat") {
  console.log(animal.indoor); // ✓ TypeScript knows this exists
}

Template Literal Types

V6 generates TypeScript template literal types from JSON Schema pattern constraints where the pattern is a simple prefix/suffix match:

// JSON Schema with pattern
{ "type": "string", "pattern": "^user_" }

// v6 generates:
type UserId = `user_${string}`;
// Instead of v5's: type UserId = string;

CLI and API Improvements

The CLI in v6 supports watching schema files for changes and auto-regenerating TypeScript types, making it practical to keep types in sync with evolving schemas during development. The programmatic API adds async support and streaming output for large schema collections.
