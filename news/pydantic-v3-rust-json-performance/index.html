Pydantic v3 Rewrites JSON Core in Rust: 3-8x Faster Validation and Serialization

Published: 2026-04-28 · 6 min read

Pydantic v3 is out, and it marks the most significant performance leap in the library's history. Building on the Rust foundation introduced in Pydantic v2 (via pydantic-core), v3 completes the migration by rewriting the JSON serialization and deserialization pipeline entirely in Rust — delivering 3-8x faster validation and up to 12x faster JSON serialization compared to v2, with zero breaking changes to the Python API.

The Architecture Shift

Pydantic v2 introduced pydantic-core, a Rust extension module handling validation logic, but JSON serialization still went through Python's standard json module for the final encoding step. Pydantic v3 eliminates this Python bottleneck entirely:

from pydantic import BaseModel
from datetime import datetime
from typing import Optional

class Order(BaseModel):
    order_id: str
    user_id: int
    items: list[dict]
    total: float
    created_at: datetime
    notes: Optional[str] = None

order = Order(
    order_id="ord_abc123",
    user_id=42,
    items=[{"sku": "A1", "qty": 2}, {"sku": "B3", "qty": 1}],
    total=149.99,
    created_at=datetime.now(),
)

# Pydantic v3: entire pipeline is Rust — Python never touches the JSON bytes
json_str = order.model_dump_json()
# ~8x faster than Pydantic v2 for this model

The new pydantic-core v3 serializer bypasses Python's object graph entirely for model_dump_json() — it reads the model's field values directly from the Rust-side representation and writes JSON bytes without any Python string intermediaries.

The Pydantic team ran benchmarks across a range of real-world model sizes, comparing v3 against v2 and v1 on Python 3.13:

The gains are most dramatic for serialization (converting Python models back to JSON), where the v3 Rust path is consistently 7-13x faster than v2. For high-throughput FastAPI endpoints returning large model lists, this translates directly into increased request throughput and reduced CPU cost.

New: Streaming JSON Serialization

Pydantic v3 introduces a new model_dump_json_stream() method for large models and lists, enabling chunk-by-chunk JSON output without materializing the full string in memory:

from pydantic import BaseModel
from typing import Iterator

class Product(BaseModel):
    id: str
    name: str
    description: str
    price: float
    attributes: dict

def stream_products(products: list[Product]) -> Iterator[bytes]:
    # Yields JSON chunks — no full string in memory
    yield b'{"products":['
    for i, product in enumerate(products):
        if i > 0:
            yield b','
        yield from product.model_dump_json_stream()
    yield b']}'

# FastAPI / Starlette streaming response
from fastapi.responses import StreamingResponse
return StreamingResponse(
    stream_products(large_product_list),
    media_type="application/json"
)

For APIs returning thousands of records, streaming reduces time-to-first-byte by eliminating the full serialization step before the response starts. Memory usage for a 10,000-item list drops from ~180MB to under 5MB with streaming enabled.

JSON Schema Generation Improvements

Pydantic v3 also overhauls its JSON Schema output to fully support Draft 2020-12. Previous versions generated Draft 7-compatible schemas with vendor extensions; v3 generates clean, standard 2020-12 documents:

from pydantic import BaseModel
from pydantic.json_schema import model_json_schema

class Address(BaseModel):
    street: str
    city: str
    country: str
    postal_code: str

class User(BaseModel):
    id: int
    name: str
    email: str
    address: Address
    tags: list[str] = []

schema = model_json_schema(User, mode="validation")
# Output: JSON Schema 2020-12
# {
#   "$schema": "https://json-schema.org/draft/2020-12/schema",
#   "type": "object",
#   "properties": { ... },
#   "$defs": { "Address": { ... } }
# }

The updated schema generator correctly handles recursive models, discriminated unions, Annotated types, and custom validators — all producing standard-compliant JSON Schema that works with any validator library.

AI and LLM Workflow Performance

The performance gains are particularly significant for AI pipelines where Pydantic is used to parse and validate LLM structured outputs. With tools like LangChain, LlamaIndex, and instructor all using Pydantic for output parsing, the v3 improvements compound across entire agent workflows:

  • instructor: structured output parsing from GPT/Claude/Gemini — 4x faster round-trip
  • LangChain output parsers: Pydantic-based chain outputs — 3x faster
  • FastAPI AI endpoints: request validation + response serialization — combined 6x improvement
  • Batch processing: validating thousands of LLM responses — most dramatic gains

Upgrading to Pydantic v3

The Pydantic team maintained strict backward compatibility. Most codebases can upgrade with just pip install "pydantic>=3.0". Key changes to be aware of:

  • model_dump_json() now returns bytes instead of str — use .decode() if you need a string
  • JSON Schema output now uses 2020-12 format — update validators consuming the schema if needed
  • model_json_schema() parameter by_alias now defaults to True
  • Custom __get_validators__ is removed — migrate to @classmethod __get_pydantic_core_schema__

A migration guide and automated pydantic-migrate CLI tool are available for larger codebases with custom validators. For the vast majority of users, the upgrade is a single command and immediate performance gains.
