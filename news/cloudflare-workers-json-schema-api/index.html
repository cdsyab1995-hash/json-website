Cloudflare Workers v3 Launches Native JSON Schema Validation at the Edge

Home / News / Cloudflare Workers v3
Published: April 26, 2026 · 5-6 min read

Cloudflare Workers v3 has shipped with a major addition that edge developers have been requesting for years: native JSON Schema validation built directly into the Workers runtime. The new cf.schema.validate() API provides Draft 2020-12 compliant validation with zero cold start overhead and global distribution built in.

Why Edge-Native Schema Validation Matters

For years, validating JSON payloads at the edge required either bundling Ajv or another schema validator into your Worker bundle — adding 15-40KB of JavaScript and measurable cold start latency. With Workers v3, schema validation is compiled into the V8 isolate itself, meaning it's available instantly on every request, anywhere in the world.

The impact is significant for API-first architectures:

  • 15-40KB saved in bundle size per Worker
  • Zero cold start overhead — validation is part of the runtime baseline
  • Draft 2020-12 full support including dynamic references ($dynamicRef) and vocabularies
  • Sub-millisecond validation for typical API payloads (under 10KB)

The Workers Schema API

The new API is accessed via the global cf object, consistent with other Cloudflare-specific runtime APIs:

export default {
  async fetch(request, env, ctx) {
    const payload = await request.json();
    
    const result = cf.schema.validate(payload, {
      type: "object",
      properties: {
        userId: { type: "string", format: "uuid" },
        action: { type: "string", enum: ["create", "update", "delete"] },
        data: { type: "object" }
      },
      required: ["userId", "action"]
    });

    if (!result.valid) {
      return new Response(JSON.stringify({
        error: "validation_failed",
        details: result.errors
      }), { status: 400 });
    }

    // Process validated payload...
    return new Response("OK");
  }
};

The cf.schema.validate() function returns a result object with valid: boolean and an errors[] array containing detailed error objects with instancePath, schemaPath, and human-readable message fields.

Integration with Cloudflare API Shield

Workers v3 schema validation integrates seamlessly with Cloudflare's API Shield product. You can now define a schema once and use it both for runtime validation in Workers and for API Shield's managed DDoS and abuse protection at the network edge.

// Export schema for API Shield
export const schema = {
  type: "object",
  properties: {
    userId: { type: "string", format: "uuid" },
    action: { type: "string", enum: ["create", "update", "delete"] },
    data: { type: "object" }
  },
  required: ["userId", "action"]
};

API Shield can import this exported schema to enforce validation before requests even reach your Worker — providing defense in depth with zero additional latency for blocked requests.

Migrating from Ajv

If you're currently using Ajv in your Workers, the migration path is straightforward. The main differences are:

  • Remove ajv and ajv-formats from your package.json
  • Replace import Ajv from 'ajv' with cf.schema.validate()
  • Error handling changes slightly — result.errors replaces Ajv's errors property on the validate function return
  • Schemas are compiled once per Worker instance, so no need for manual compile() calls

Availability

Cloudflare Workers v3 is now generally available. The Schema API is enabled by default on all Workers — no plan upgrade required. Documentation is available at developers.cloudflare.com/workers/runtime-apis/schema.
