Node.js 24 Ships Built-in JSON Schema Validation: Native Schema Enforcement Without Dependencies

Published: 2026-04-29 · 4-5 min read

Node.js 24 includes a new standard library module: node:schema, providing native JSON Schema Draft 2020-12 validation without requiring any third-party packages. For API servers, request validation, and configuration parsing — some of the most common Node.js use cases — this eliminates the need to install and bundle Ajv, Zod, or similar libraries for basic schema enforcement.

The New node:schema Module

The node:schema module exposes a single primary API: the Validator class that compiles a JSON Schema once and validates data repeatedly against it. The interface is intentionally minimal and mirrors the best-of-breed patterns from Ajv:

import { Validator } from 'node:schema';

const UserSchema = {
  type: 'object',
  properties: {
    id: { type: 'string', format: 'uuid' },
    name: { type: 'string', minLength: 1, maxLength: 100 },
    email: { type: 'string', format: 'email' },
    age: { type: 'integer', minimum: 0, maximum: 150 }
  },
  required: ['id', 'name', 'email'],
  additionalProperties: false
};

const validator = new Validator(UserSchema);

// Returns { valid: true } or { valid: false, errors: [...] }
const result = validator.validate(requestBody);

if (!result.valid) {
  console.error('Validation errors:', result.errors);
}

Performance: Native vs. Third-Party

The node:schema validator is implemented in C++ within the Node.js core, using a pre-compiled validation engine that avoids JavaScript parsing overhead. Benchmarks from the Node.js performance team show:

  • Simple object validation: node:schema is 2.1x faster than Ajv 8, 4.8x faster than Zod v4
  • Complex nested schema (10 levels deep): node:schema is 1.8x faster than Ajv 8
  • Bundle size impact: Zero — no bytes added to your application bundle

The performance advantage is most pronounced in high-throughput API servers where request validation runs on every incoming request. For a server handling 10,000 requests/second with 50μs validation time, switching to node:schema can recover ~500ms of CPU time per second.

Supported JSON Schema Features

Node.js 24's initial implementation supports the core JSON Schema Draft 2020-12 vocabulary:

  • All primitive types: string, number, integer, boolean, null, array, object
  • String constraints: minLength, maxLength, pattern, format (email, uuid, uri, date, date-time)
  • Numeric constraints: minimum, maximum, exclusiveMinimum, exclusiveMaximum, multipleOf
  • Object keywords: properties, required, additionalProperties, minProperties, maxProperties
  • Array keywords: items, minItems, maxItems, uniqueItems
  • Composition: allOf, anyOf, oneOf, not
  • References: $defs and local $ref

Cross-document $ref (referencing external schema files) is planned for Node.js 24.1 but not included in the initial release.

Integration with Express and Fastify

For Express users, integrating node:schema into request validation is straightforward:

import express from 'express';
import { Validator } from 'node:schema';
import { CreateUserSchema } from './schemas.js';

const app = express();
const validateUser = new Validator(CreateUserSchema);

app.post('/users', (req, res, next) => {
  const result = validateUser.validate(req.body);
  if (!result.valid) {
    return res.status(422).json({ errors: result.errors });

Fastify users will benefit even more directly: Fastify already uses JSON Schema for route validation (via Ajv under the hood). The Fastify team has announced an official adapter that swaps Ajv for node:schema with a single configuration option, expected in Fastify v5.1.

What This Doesn't Replace

It's important to clarify that node:schema is a JSON Schema validator, not a TypeScript schema library like Zod or Valibot. It validates runtime data but provides no TypeScript inference — you won't get typed output from validation. Libraries like Zod remain the right choice when you need tight TypeScript integration, schema composition, or the ability to derive TypeScript types from your schemas. node:schema targets the pure runtime validation use case where performance and zero-dependency footprint matter most.
