Python orjson 4.0: Faster JSON for Data Science

Published: 2026-05-04 · 2 min read · 312 words

orjson 4.0: Built for Data Science

Orjson, the fastest Python JSON library, released version 4.0 with features specifically designed for data science workflows. The new release adds native support for serializing pandas DataFrames, NumPy arrays, and Python dataclasses with significantly improved performance.

Data science applications often need to serialize large datasets to JSON for storage, API transmission, or inter-process communication. Orjson 4.0's new DataFrame serialization support handles the conversion automatically, producing compact JSON representations of tabular data that can be efficiently transmitted and reconstructed.

The new version maintains orjson's position as the fastest Python JSON library, with serialization speeds approximately 10x faster than the standard library json module and deserialization speeds approximately 3x faster. For applications that process large volumes of JSON data, these improvements translate directly to reduced processing time and lower computational costs.

New Features for Data Science Workflows

Beyond raw performance, orjson 4.0 introduces features that simplify data science workflows. Native NumPy array serialization preserves the full structure and type information of arrays, enabling round-trip serialization that maintains precision for floating-point data.

Pandas DataFrame serialization handles the column-based structure of DataFrames efficiently, producing JSON that represents the data in a format that's both human-readable and easy to reconstruct. The serialization is configurable, allowing users to choose between different representations for dates, missing values, and complex types.

The new dataclass serialization support automatically handles Python dataclasses with proper type preservation, reducing the boilerplate code required for JSON serialization in data processing applications.

Installation and Migration

Orjson 4.0 is available via pip and conda-forge. The package includes pre-compiled wheels for common platforms, ensuring fast installation without requiring a Rust compiler. Users upgrading from orjson 3.x will find that existing code continues to work without modification, as the API remains fully compatible.

The migration for data science applications may require updating serialization calls to take advantage of new features, but the library gracefully handles missing optional dependencies, falling back to standard behavior when specialized serializers aren't available.
