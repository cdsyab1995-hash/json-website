Firefox 135 DevTools Major Overhaul: Visual JSON Schema Explorer and Live Validation

Published: 2026-04-26 · 5 min read

Firefox 135 has shipped one of the most ambitious DevTools updates in the browser's history. The new JSON handling panel introduces a visual JSON Schema explorer, schema-driven autocomplete, and live validation at 60fps — all directly in the DevTools JSON viewer. This closes the gap that has existed between browser DevTools and dedicated JSON editing tools.

Visual JSON Schema Explorer

When a JSON file or API response is associated with a JSON Schema (via a $schema field or a linked schema in DevTools settings), Firefox 135 displays a split-panel view: the JSON data on the left, a navigable schema tree on the right.

The schema explorer shows the full type hierarchy — object properties, array item types, enum values, format constraints, and default values — all collapsible and searchable. Clicking any property in the schema explorer highlights its corresponding value in the JSON data.

This is particularly valuable for large API responses where understanding the schema was previously a manual, time-consuming process of cross-referencing documentation.

Schema-Driven Autocomplete

Perhaps the most practical new feature: when editing JSON in the DevTools JSON editor, Firefox 135 provides autocomplete suggestions driven by the active schema. As you type, the editor shows valid property names, enum values, and type constraints based on the schema's properties, oneOf, and enum definitions.

For nested objects, the autocomplete context updates as you navigate — suggestions are scoped to the current object type, not the entire schema. This makes writing complex nested JSON significantly faster and reduces schema violations.

Live Validation at 60fps

Previous browser DevTools JSON editors validated on blur or on manual trigger. Firefox 135 validates continuously using a lightweight Rust-based validator (compiled to WebAssembly), maintaining 60fps editing performance even for multi-megabyte JSON documents.

Validation errors appear as inline annotations in the editor — red underlines on invalid values with tooltip explanations — and also in the Console as structured error objects. The error format follows the JSON Schema Validation Output Standard (ISO/IEC 24017), making it easy to process programmatically.

New JSON Editing Features

  • Multi-cursor editing — Select multiple properties and rename them in one action
  • JSON diff viewer — Compare two JSON documents side-by-side with inline diff highlighting
  • Schema inference — If no schema is present, Firefox can infer a Draft 2020-12 schema from the JSON data and offer it as a starting point
  • Import/export schemas — Link external schema files to the DevTools workspace
  • Copy as cURL — Right-click any JSON editor content to generate a cURL command with the JSON as the request body

Performance: How Fast Is It?

Mozilla's benchmarks on the new JSON editing panel:

OperationFirefox 134Firefox 135
Parse 10MB JSON340ms85ms
Type-ahead latency8ms2ms
Validation (1MB, complex schema)120ms15ms
Scroll smoothness45fps60fps

The dramatic improvements come from the Rust/WebAssembly JSON parsing engine, which is now shared across the DevTools JSON viewer, the network response panel, and the Storage inspector.

How to Enable

Firefox 135 is available now. The new JSON tools are enabled by default. Open DevTools (F12 or Cmd+Option+I), navigate to any network request with a JSON response, or use the new JSON Editor panel (added to the default panel layout in Firefox 135). Schema linking is available in DevTools Settings → JSON → Linked Schemas.
