Bun 2.0 Ships Custom JSON Serialization Engine: 5x Faster Than V8's JSON.stringify

Published: 2026-04-28 · 3-4 min read

Bun 2.0 has arrived with one of the most impactful performance improvements for JSON-heavy server applications: a custom JSON serialization engine written in Zig with SIMD acceleration that replaces V8's built-in JSON.stringify. Benchmarks from the Bun team show 5x throughput improvement for large object serialization, which directly translates to faster API response times for REST and JSON-RPC servers running on Bun.

Why Replace V8's JSON.stringify?

V8's JSON.stringify implementation is general-purpose and optimized for browser use cases, where JSON serialization is occasional and typically involves small objects. For server workloads — where an API server might serialize thousands of response objects per second — V8's approach leaves significant performance on the table. By dropping into Zig with direct SIMD instruction access, Bun can exploit parallel string escaping, numeric-to-string conversion batching, and cache-efficient memory layouts that V8's GC-managed approach can't match.

Benchmark Numbers

The Bun team's benchmark comparing JSON.stringify performance across runtimes on a 10KB nested object:

  • Bun 2.0: 2.1M serializations/second
  • Node.js 24: 420K serializations/second
  • Deno 2.2: 380K serializations/second
  • Bun 1.2 (baseline): 410K serializations/second

The improvement is most dramatic for larger objects (50KB+), where SIMD string escaping provides the greatest parallel speedup. For small objects under 1KB, the improvement is more modest at 1.5-2x.

100% Spec-Compatible

Bun's custom serializer passes the full ECMAScript spec test suite for JSON.stringify, including edge cases like toJSON() methods, replacer functions, space indentation, circular reference detection, and special numeric values (NaN, Infinity → null). The behavior is identical to V8 for all standard use cases — it's a drop-in replacement that requires no code changes.

Bun 2.0 Additional Changes

Beyond JSON serialization, Bun 2.0 ships several other noteworthy improvements: native fetch() streaming support, a built-in SQLite ORM layer, and hot module replacement for server-side code. For teams building JSON APIs, the combination of faster serialization and streaming support makes Bun 2.0 a compelling upgrade.
