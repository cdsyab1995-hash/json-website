Streaming JSON APIs with NDJSON: A Practical Guide for 2026

Published: 2026-04-29 · 7-9 min read

Traditional REST APIs return complete JSON responses at once. For large datasets or real-time feeds, this creates latency problems — users wait for the entire response before seeing any data. NDJSON (Newline-Delimited JSON) solves this by sending JSON objects one per line, separated by newlines. Each line is a complete, valid JSON object that can be processed immediately. This pattern is powering modern AI APIs, real-time dashboards, and high-performance data pipelines.

Prerequisites

For this tutorial, you'll need: - Node.js 18+ (for native streaming support) - Basic understanding of HTTP and JSON - Familiarity with async iterators (or willingness to learn!) Install these packages: 
npm install express ndjson
npm install --save-dev typescript @types/node
## Server-Side: Streaming NDJSON Responses Express with ndjson middleware: 
import createNdjsonStream from 'ndjson';
import { Router } from 'express';

const router = Router();

// Stream thousands of records efficiently
router.get('/api/events/stream', (req, res) => {
  res.setHeader('Content-Type', 'application/x-ndjson');
  res.setHeader('Transfer-Encoding', 'chunked');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const transform = createNdjsonStream.serialize();

  transform.pipe(res);

  // Stream data as it becomes available
  let count = 0;
  const interval = setInterval(() => {
    const record = {
      id: count++,
      timestamp: new Date().toISOString(),
      event: 'user_action',
      data: { userId: `user_${count}`, action: 'click' }
    };

    transform.write(record);

    if (count >= 1000) {
      clearInterval(interval);
      transform.end();
    }
  }, 10); // 100 records/second
});
## Client-Side: Consuming NDJSON Streams 
// Using native fetch with ReadableStream (Node 18+)
async function* streamNdjson(url: string) {
  const response = await fetch(url);

  if (!response.body) {
    throw new Error('Streaming not supported');
  }

  const reader = response.body
    .pipeThrough(new TextDecoderStream())
    .pipeThrough(createNdjsonParseStream());

  for await (const line of reader) {
    yield line;
  }
}

// Usage: Process stream as data arrives
for await (const record of streamNdjson('/api/events/stream')) {
  console.log('Received:', record.timestamp);
  updateDashboard(record); // UI updates immediately
}
Browser Support: ReadableStream is supported in all modern browsers. For older browsers, use a polyfill or the stream-consuming library.

Advanced Tips

## Advanced: Error Handling and Resilience NDJSON streams can fail midway. Here's how to handle it: 
class ResilientNdjsonStream {
  constructor(url, options = {}) {
    this.url = url;
    this.options = options;
    this.retryDelay = options.retryDelay || 1000;
    this.maxRetries = options.maxRetries || 3;
  }

  async *stream() {
    let retries = 0;

    while (retries < this.maxRetries) {
      try {
        const response = await fetch(this.url);
        const stream = response.body
          .pipeThrough(new TextDecoderStream())
          .pipeThrough(createNdjsonParseStream());

        for await (const record of stream) {
          retries = 0; // Reset on successful read
          yield record;
        }

        return; // Stream ended normally
      } catch (error) {
        retries++;
        console.warn(`Stream failed, retry ${retries}/${this.maxRetries}`);

        if (retries < this.maxRetries) {
          await this.delay(this.retryDelay * retries);
        }
      }
    }

    throw new Error('Max retries exceeded');
  }

  delay(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}
