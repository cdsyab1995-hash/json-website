Browser DevTools 2026: Every Major Browser Ships Built-in JSON Schema Explorer and Live Validation

Published: 2026-04-28 · 3-4 min read

2026 marks the year browser DevTools became genuinely JSON Schema-aware. Chrome 135, Firefox 136, and Safari 18.2 have all shipped built-in JSON Schema exploration and live validation in their network inspector panels — no extensions required. For developers debugging API integrations, this eliminates the need to copy-paste responses into separate validation tools.

What's New in the Network Panel

The new JSON tooling appears in the network panel's response inspector. When a response body is valid JSON, browsers now offer:

  • Schema Explorer: A tree view that infers the JSON structure and displays it as an annotated schema, showing inferred types, optional fields, and array shapes
  • Schema Validation: If the request URL matches a registered API spec (via Service Worker registration or a new Link: rel="describedby" header), the response is automatically validated against the associated JSON Schema, with violations highlighted in the response tree
  • Copy as JSON Schema: Right-click any JSON response to generate a JSON Schema draft from it instantly

Chrome 135's Implementation

Chrome's approach is the most integrated: it reads Link: <schema.json>; rel="describedby" response headers and automatically fetches and applies the referenced JSON Schema. Any property that violates the schema is highlighted in red in the response viewer, with a tooltip explaining the violation. The feature works on all HTTPS responses and supports JSON Schema Draft 2020-12.

Firefox's Standalone Schema Validator

Firefox 136 takes a slightly different approach, adding a dedicated "Schema" tab in the network response panel where developers can paste or upload a JSON Schema and validate the selected response against it on demand. This decoupled approach is more manual but doesn't require server-side header configuration, making it easier to use with third-party APIs.

Safari's Type Inference View

Safari 18.2 focuses on schema inference rather than validation: it automatically generates a readable type description for any JSON response, displayed as a collapsible type tree alongside the raw JSON. While it doesn't validate against external schemas yet, the inference view is useful for understanding unfamiliar API response shapes at a glance.
