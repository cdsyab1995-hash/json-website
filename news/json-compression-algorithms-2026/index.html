New Compression Algorithms Excel at JSON Data

Published: 2026-05-04 · 2 min read · 312 words

Generic Compression Falls Short for JSON

Gzip and Brotli compress JSON effectively, but they are general-purpose algorithms that do not understand JSON structure. JSON-aware compression can achieve smaller sizes by exploiting patterns specific to JSON: repeated keys, structural characters, number formats. New algorithms like SOUP (Structured Optimization for Useful Packets) specifically target JSON and similar structured text, achieving 20-40% better compression than gzip on typical JSON payloads.

CBOR: Binary JSON for Space-Efficient Storage

Concise Binary Object Representation (CBOR) encodes JSON-like data in binary form, typically reducing payload size by 30-50%. Unlike text JSON, CBOR uses binary encoding for numbers, strings, and arrays. This makes CBOR ideal for IoT devices with limited bandwidth, mobile apps where payload size affects battery life, and storage systems where space matters. Libraries exist for every major language, making CBOR a practical choice.

Random Access in Compressed JSON

A limitation of compressed JSON is losing random access—you must decompress everything to read a specific element. Newer formats address this. UBJSON (Universal Binary JSON) can include size and type markers that enable skipping to specific array indices. Indexes can be added to NDJSON files to enable random access to specific line ranges. Some systems pre-compute indexes during compression that enable efficient queries without full decompression.

Choosing the Right Compression Strategy

The right approach depends on your constraints. For HTTP APIs where bandwidth is the bottleneck, Brotli compression (widely supported by browsers and servers) is the easiest win—enable it and you are done. For extreme size reduction, consider CBOR or specialized algorithms. For mixed workloads with both compression and query needs, look at formats with index support. Profile your actual payload characteristics before committing to a format change.

Future of JSON Serialization Formats

The trend toward specialized formats will continue as use cases diversify. Expect more format options targeting specific domains: time-series data, graph data, spatial data. Schema-aware compression will improve as schemas become more common and machine-readable. Hybrid approaches (partly human-readable, partly binary) may emerge. Web Standards are considering binary JSON alternatives at the standards level.
