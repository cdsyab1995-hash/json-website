JSON to CSV: Convert JSON Data to Spreadsheet Format

Published: 2026-05-04 · 5 min read · 952 words

Why Convert JSON to CSV?

CSV remains the universal format for data exchange with spreadsheets, databases, and analytics tools. While JSON is the standard for APIs and modern data interchange, the business world runs on Excel, Google Sheets, and SQL databases—all of which work natively with CSV.

The need to convert JSON to CSV arises constantly: an API returns JSON, but your manager wants to analyze the data in Excel; a database export is in JSON format, but you need to import it into a legacy system that only accepts CSV; a webhook delivers JSON payloads, but you want to analyze patterns in a spreadsheet.

Understanding how to convert JSON to CSV efficiently is an essential skill for anyone working with data. The conversion isn't always straightforward—JSON's nested structures and varied types don't map directly to CSV's flat row-column format—but there are established techniques for handling these challenges.

The Challenge: JSON and CSV Are Fundamentally Different

JSON and CSV represent data differently, and understanding these differences is crucial for successful conversion.

JSON supports nested objects, arrays of objects, heterogeneous types, and null values. A single JSON document can represent complex hierarchical relationships between data. CSV is a flat grid: each row represents one record, and each column represents one attribute. There's no inherent support for nesting or hierarchy.

Arrays in JSON pose the biggest conversion challenge. If a JSON object has an array property, how should it appear in a flat CSV row? Options include joining array values with a delimiter, exploding the array into multiple rows (one per array item), or ignoring the array entirely.

Missing fields require careful handling. If some JSON objects have a "status" property and others don't, how do you represent this in CSV? The standard approach is to use empty cells or a placeholder like "N/A" for missing values.

Type information is lost in CSV. A JSON value of 42 (number) and "42" (string) both appear as "42" in CSV. This matters when the CSV is processed—some systems will treat the cell as a number, others as text.

Flattening Nested JSON for CSV Export

The most common approach to converting JSON to CSV is to flatten the JSON structure first, then export as CSV. Flattening converts nested objects into dot-notation property names.

For example, a JSON object like {"name": "Alice", "address": {"city": "NYC", "zip": "10001"}} flattens to {"name": "Alice", "address.city": "NYC", "address.zip": "10001"}. This creates additional columns in the CSV: name, address.city, and address.zip.

Array handling during flattening requires a decision: either join array values into a single cell (using a delimiter like semicolon) or expand the JSON object into multiple rows (one per array item). The choice depends on your use case and what analysis you plan to perform.

Flattening can be implemented in any programming language. JavaScript and Python both have libraries that handle flattening automatically. For JavaScript, you can use flat (a built-in method) or libraries like flat or lodash. For Python, pandas.json_normalize() handles flattening automatically.

Converting JSON Arrays to CSV Rows

When JSON contains an array of objects, each object typically becomes a row in the CSV. This is the most straightforward case and works well for most API responses.

Consider an API response like {"users": [{"name": "Alice", "email": "alice@example.com"}, {"name": "Bob", "email": "bob@example.com"}]}. The conversion process extracts the array, uses the object keys as column headers, and creates a row for each object: name, email; Alice, alice@example.com; Bob, bob@example.com.

Nested arrays require more thought. If each user has an "orders" array, you need to decide: should the orders array become multiple rows (one per order, with user data repeated), should it be joined into a single cell, or should it be ignored? The right choice depends on your analysis needs.

Empty arrays and null values need handling. Empty arrays might become empty cells, or they might cause the row to be skipped entirely. Null values typically become empty cells or a placeholder like "N/A".

Handling Edge Cases in JSON to CSV Conversion

Real-world JSON often contains edge cases that require special handling.

Special characters in values need escaping. If a string value contains a comma, the CSV will be malformed unless the value is quoted. If it contains quotes, those quotes need to be escaped according to CSV rules. Most CSV libraries handle this automatically.

Unicode and international characters should work in modern CSV files with UTF-8 encoding, but some older spreadsheet applications have issues with certain characters. Testing with your actual data is important.

Date formats should be considered. JSON dates are often ISO 8601 strings. When converting to CSV, you might want to format dates for the target application—Excel-friendly formats, locale-specific formats, or Unix timestamps depending on what the CSV will be used for.

Very large JSON files may not fit in memory during conversion. For files larger than available RAM, streaming conversion is necessary. Read the JSON file incrementally, process each record, and write to the CSV file without holding the entire dataset in memory.

Tools for JSON to CSV Conversion

Many tools exist for converting JSON to CSV, from online converters to programming libraries.

Online converters are convenient for one-time conversions or small files. You paste JSON, specify options like delimiter handling and array expansion strategy, and download the CSV. Best for occasional use or quick tasks.

Command-line tools like jq can convert JSON to CSV with the right queries. jq has a CSV output format that handles basic conversions. For complex flattening, you might need to combine jq with other tools.

Programming libraries provide the most control. Python's pandas library can read JSON and write CSV with extensive options for handling edge cases. JavaScript's Papa Parse library handles CSV parsing and generation with good edge case handling.

Spreadsheet applications like Excel and Google Sheets can sometimes import JSON directly, though the results vary depending on the JSON structure. This is worth trying for quick analyses, but programming libraries give you more control for production workflows.
