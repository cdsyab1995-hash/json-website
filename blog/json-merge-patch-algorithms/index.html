JSON Merge Patch: A Simple Algorithm for Partial Updates

Published: 2026-05-05 · 2 min read · 461 words

The Problem: How Do You Update Part of a JSON Document?

When building APIs, you often need to update individual fields in an object without touching the rest. The naive approach is to send the entire object with just one field changed, but this requires the client to track state, increases bandwidth, and creates race conditions when multiple clients update the same object simultaneously. JSON Merge Patch (RFC 7386) provides a simple solution: send only the fields you want to change, and the server merges them into the existing document. Deleted fields are represented by null in the patch.

How JSON Merge Patch Works

JSON Merge Patch is beautifully simple. The patch is itself a JSON object. To merge a patch into a target document, you recursively apply each property: if the patch has a value that is not null, set that property in the target to the patch value. If the patch has null, remove that property from the target. If the patch has an array, replace the target array entirely. That is it—no operations, no paths, no tests. The simplicity makes it easy to implement, debug, and understand.

When Merge Patch Falls Short

Merge patch simplicity is also its limitation. You cannot append to an array; you can only replace it. You cannot update a deeply nested property without including the entire parent structure. You cannot conditionally apply changes based on the current value. For these scenarios, JSON Patch (RFC 6902) provides a more powerful alternative. JSON Patch defines operations like add, remove, replace, move, copy, and test, with JSON Pointers to identify the exact location to modify.

Implementing Merge Patch in JavaScript

Implementing merge patch is straightforward. A recursive function that handles objects and primitives covers most cases. For objects, iterate over patch properties: if the value is null, delete the property from the target; otherwise, recursively merge or replace. For arrays, simply replace. The tricky part is handling the case where the target does not have a property that the patch wants to update—create it. Many libraries exist if you prefer not to roll your own.

Using Merge Patch in REST APIs

The HTTP PATCH method was designed for partial updates, and JSON Merge Patch is one of the most common formats for PATCH bodies. Set the Content-Type to application/merge-patch+json. Your API documentation should specify that clients should send only the fields they want to update. Document that null means delete. Consider providing optimistic locking (ETags) to handle concurrent updates. If the client sends a patch based on stale data, you can detect the conflict and return 409 Conflict.

Testing Merge Patch Implementations

Test your merge patch implementation thoroughly. Create a test suite covering: adding new properties to existing objects, overwriting existing property values, deleting properties by setting them to null, nested object merging, array replacement, null patches (delete everything), empty patches (modify nothing), handling missing target properties, and type mismatches. Edge cases like merging into null should all be specified and tested.
