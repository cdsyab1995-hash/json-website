JSON Schema Validation in Practice: Building Reliable APIs with Draft 2020-12

Published: 2026-05-01 · 4-6 min read

TypeScript types, Pydantic models, and Java classes validate data at compile time or during deserialization. But when data comes from external sources — API requests, webhooks, third-party integrations, or user input — you need runtime validation. JSON Schema is the standard for describing and validating JSON structure, types, and constraints.

Draft 2020-12 is the latest JSON Schema specification. Key improvements over Draft-07 include: prefixItems for tuple-type arrays (replacing items as an array), $dynamicAnchor and $dynamicRef for recursive schemas that work across file boundaries, and boolean schemas where true means 'anything is valid' and false means 'nothing is valid'.

Basic Schema Structure

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "email": {
      "type": "string",
      "format": "email",
      "description": "User's email address"
    },
    "password": {
      "type": "string",
      "minLength": 8,
      "pattern": "^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).+$",
      "description": "Must contain uppercase, lowercase, and a digit"
    },
    "name": {
      "type": "string",
      "minLength": 1,
      "maxLength": 100
    },
    "age": {
      "type": "integer",
      "minimum": 13,
      "maximum": 120
    }
  },
  "required": ["email", "password", "name"],
  "additionalProperties": false
}

Validation in JavaScript with Ajv

import Ajv from 'ajv';
import addFormats from 'ajv-formats';

const ajv = new Ajv({ allErrors: true });
addFormats(ajv);

// Define schema
const schema = {
  type: 'object',
  properties: {
    email: { type: 'string', format: 'email' },
    age: { type: 'integer', minimum: 0, maximum: 150 }
  },
  required: ['email', 'age'],
  additionalProperties: false
};

// Compile validator
const validate = ajv.compile(schema);

// Test data
const valid = { email: 'user@example.com', age: 25 };
const invalid = { email: 'not-an-email', age: -5 };

console.log(validate(valid));    // true
console.log(validate(invalid));  // false
console.log(validate.errors);
// [
//   { keyword: 'format', params: { format: 'email' }, ... },
//   { keyword: 'minimum', params: { minimum: 0 }, ... }
// ]

Express.js Middleware Example

import express from 'express';
import Ajv from 'ajv';
import addFormats from 'ajv-formats';

const ajv = new Ajv({ allErrors: true });
addFormats(ajv);

// Create validation middleware
function validateBody(schema) {
  const validate = ajv.compile(schema);
  return (req, res, next) => {
    if (!validate(req.body)) {
      return res.status(400).json({
        success: false,
        error: {
          code: 'VALIDATION_ERROR',
          message: 'Request body validation failed',
          details: validate.errors.map(e => ({
            field: e.instancePath || e.params.missingProperty,
            rule: e.keyword,
            message: e.message
          }))
        }
      });
    }
    next();
  };
}

// Usage in routes
app.post('/api/users',
  validateBody(userSchema),
  async (req, res) => {
    // req.body is guaranteed to match the schema
    const user = await createUser(req.body);
    res.json({ success: true, data: user });
  }
);

Validation in Python

import jsonschema
from jsonschema import validate, ValidationError

# Define schema
schema = {
    "type": "object",
    "properties": {
        "email": {"type": "string", "format": "email"},
        "name": {"type": "string", "minLength": 1},
        "tags": {
            "type": "array",
            "items": {"type": "string"},
            "minItems": 1,
            "maxItems": 10,
            "uniqueItems": True
        }
    },
    "required": ["email", "name"],
    "additionalProperties": False
}

# Validate data
try:
    validate(instance=data, schema=schema)
    print("Valid!")
except ValidationError as e:
    print(f"Invalid: {e.message}")
    print(f"Field: {e.json_path}")
    print(f"Rule: {e.validator}")  # e.g., 'minLength', 'type'

Common Validation Patterns

// Schema with conditional requirements
{
  "type": "object",
  "properties": {
    "role": {
      "type": "string",
      "enum": ["admin", "user", "guest"]
    },
    "permissions": {
      "type": "array",
      "items": { "type": "string" }
    }
  },
  "required": ["role"],
  "if": {
    "properties": { "role": { "const": "admin" } }
  },
  "then": {
    "required": ["permissions"]
  }
}
// Admins MUST have permissions; other roles don't need them
{
  "type": "object",
  "properties": {
    "version": { "type": "string", "const": "2.0" },
    "data": {
      "type": "object",
      "properties": {
        "id": { "type": "string", "pattern": "^usr_[a-z0-9]+$" },
        "created_at": { "type": "string", "format": "date-time" }
      },
      "required": ["id", "created_at"]
    }
  },
  "required": ["version", "data"]
}

Compile your schemas once and reuse the validator function. Ajv compiles JSON Schema into JavaScript functions that run 10-100x faster than interpreting the schema on each validation call. For high-throughput APIs, this makes a significant difference. Also consider using $ref to keep schemas DRY and compiled validators cached.

Key Takeaways

  • JSON Schema Draft 2020-12 is the current standard — use it for new projects
  • Always validate external data at runtime — types and interfaces are not enough
  • Compile validators once, reuse them — Ajv compiles to JavaScript functions for speed
  • Use allErrors: true to return all validation failures at once, not just the first
  • Conditional validation with if/then/else handles complex business rules
  • Integrate validation into API middleware to catch bad data before it hits your business logic
