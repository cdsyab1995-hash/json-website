Streaming JSON: NDJSON and JSON Lines for Large-Scale Data Processing

Published: 2026-05-02 · 4-6 min read

Standard JSON requires you to load the entire document before parsing begins — you can't parse a 10 GB JSON array incrementally. NDJSON (Newline Delimited JSON) and its alias JSON Lines (.jsonl) solve this by storing one JSON object per line. Each line is valid, self-contained JSON. You can process billions of records with constant memory usage, pipe between processes, and resume from any line.

NDJSON vs Standard JSON

// Standard JSON — must load entire file:
[
  { "id": 1, "event": "login", "user": "alice" },
  { "id": 2, "event": "purchase", "user": "bob" },
  { "id": 3, "event": "logout", "user": "alice" }
]

// NDJSON / JSON Lines — one object per line:
{ "id": 1, "event": "login", "user": "alice" }
{ "id": 2, "event": "purchase", "user": "bob" }
{ "id": 3, "event": "logout", "user": "alice" }

// Key properties:
// - Each line is independent valid JSON
// - Lines separated by \n (Unix) or \r\n (Windows)
// - No trailing comma between lines (not an array)
// - Blank lines are allowed and ignored
// - File extension: .ndjson or .jsonl

Real-World Use Cases

NDJSON is the format of choice for several high-volume scenarios: LLM streaming responses (OpenAI's streaming API sends NDJSON server-sent events), log aggregation (Fluentd, Logstash, CloudWatch all use JSON Lines), database exports (MongoDB's mongoexport outputs NDJSON by default), ETL pipelines (streaming large datasets between systems without buffering), and Kubernetes audit logs. Whenever you see Content-Type: application/x-ndjson or application/jsonl, you're working with this format.

Processing NDJSON in JavaScript

import { createReadStream } from 'fs';
import { createInterface } from 'readline';

// Process a large NDJSON file line by line
async function processNdjson(filePath) {
  const fileStream = createReadStream(filePath);
  const rl = createInterface({
    input: fileStream,
    crlfDelay: Infinity  // Handle Windows \r\n
  });

  let count = 0;
  for await (const line of rl) {
    if (!line.trim()) continue;  // Skip blank lines

    try {
      const record = JSON.parse(line);
      // Process each record independently
      await processRecord(record);
      count++;
    } catch (err) {
      console.error('Bad line:', line, err.message);
    }
  }

  console.log(`Processed ${count} records`);
}

await processNdjson('./events.ndjson');
// Read streaming NDJSON from an API endpoint
async function streamNdjsonApi(url) {
  const response = await fetch(url);
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop();  // Keep incomplete last line

    for (const line of lines) {
      if (!line.trim()) continue;
      try {
        const data = JSON.parse(line);
        handleData(data);  // Process each chunk as it arrives
      } catch { /* ignore malformed lines */ }
    }
  }
}

Processing NDJSON in Python

import json

# Generator: reads one line at a time (constant memory)
def iter_ndjson(file_path):
    with open(file_path, 'r', encoding='utf-8') as f:
        for line_num, line in enumerate(f, 1):
            line = line.strip()
            if not line:
                continue
            try:
                yield json.loads(line)
            except json.JSONDecodeError as e:
                print(f'Line {line_num}: Parse error — {e}')

# Process 10 million records without memory issues:
for record in iter_ndjson('events.ndjson'):
    process(record)

# Write NDJSON output
def write_ndjson(records, output_path):
    with open(output_path, 'w', encoding='utf-8') as f:
        for record in records:
            f.write(json.dumps(record, ensure_ascii=False) + '\n')

# Or with pandas (for analytics):
import pandas as pd
df = pd.read_json('events.ndjson', lines=True)  # Built-in support!
print(df.head())

Generating NDJSON Output

// Node.js: Generate NDJSON from database cursor
import { createWriteStream } from 'fs';

async function exportToNdjson(cursor, outputPath) {
  const stream = createWriteStream(outputPath, { encoding: 'utf-8' });

  for await (const record of cursor) {
    stream.write(JSON.stringify(record) + '\n');
  }

  stream.end();
  console.log('Export complete:', outputPath);
}

// HTTP server: Stream NDJSON response
app.get('/api/events/stream', async (req, res) => {
  res.setHeader('Content-Type', 'application/x-ndjson');
  res.setHeader('Transfer-Encoding', 'chunked');

  const cursor = db.collection('events').find();
  for await (const doc of cursor) {
    res.write(JSON.stringify(doc) + '\n');
  }
  res.end();
});

NDJSON vs JSON Array Performance

MetricJSON ArrayNDJSON
Memory usage (10M records)Loads entire file (~5GB RAM)~1 KB (streaming)
Time to first recordAfter full parse completesImmediate (line 1)
Resumable processingNo — start over on failureYes — restart from last line
Parallel processingComplex — requires splittingEasy — split at any \n
Append new recordsRequires full rewriteJust append a new line
Schema flexibilityAll records in one schemaEach line can differ

Key Takeaways

  • NDJSON / JSON Lines stores one JSON object per line — no array brackets, no commas between records
  • Streaming NDJSON enables constant-memory processing of arbitrarily large datasets
  • LLM APIs (OpenAI, Anthropic) use NDJSON streaming for progressive token delivery
  • In Node.js, use readline with createInterface for line-by-line file processing
  • In Python, use a generator with json.loads(line); pandas supports lines=True natively
  • MongoDB exports, Kubernetes audit logs, and Fluentd all use NDJSON as their default format
