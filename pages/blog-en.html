What is JSON?

JSON (JavaScript Object Notation) is a lightweight data interchange format, popularized by Douglas Crockford in 2001. It is based on a subset of JavaScript but is language-independent, with nearly all modern programming languages providing native support for JSON.

Why Learn JSON?
  • API Development: Over 95% of web APIs use JSON for data transmission
  • Configuration Files: package.json, tsconfig.json, and more
  • Data Storage: NoSQL databases, configuration files
  • Logging Systems: Structured log recording

JSON Characteristics

  • Clean & Readable: More compact than XML with clear structure
  • Easy to Parse: Built-in support in all major programming languages
  • Cross-Platform: Language-independent, exchangeable between any systems
  • Type-Safe: Supports multiple data types

JSON Basic Syntax

JSON has strict syntax rules. Violating any of them will cause parsing to fail.

Basic Structure

{
  "name": "John Smith",
  "age": 28,
  "isStudent": false,
  "skills": ["JavaScript", "Python", "SQL"],
  "address": {
    "city": "New York",
    "district": "Manhattan"
  }
}

Core Rules

Rule Correct Example Incorrect Example
Keys must use double quotes "name" name or 'name'
Strings must use double quotes "Hello" 'Hello'
No trailing commas [1, 2, 3] [1, 2, 3, ]
No comments (No comments) // comment
No undefined Use null "value": undefined

JSON Data Types

JSON supports 6 primitive data types.

1. String

{
  "message": "Hello World",
  "unicode": "Chinese and emoji",
  "escape": "newline\\n and tab\\t"
}

2. Number

{
  "integer": 42,
  "float": 3.14159,
  "negative": -17,
  "scientific": 1.5e10,
  "hex": 0xFF
}

3. Boolean

{
  "isActive": true,
  "isDeleted": false
}

4. Null

{
  "middleName": null,
  "deletedAt": null
}

5. Object

{
  "user": {
    "id": 1,
    "name": "Jane Doe",
    "email": "jane@example.com"
  }
}

6. Array

{
  "colors": ["red", "green", "blue"],
  "numbers": [1, 2, 3, 4, 5],
  "mixed": [1, "two", true, null],
  "objects": [
    {"id": 1, "name": "A"},
    {"id": 2, "name": "B"}
  ]
}

JSON vs JavaScript Objects

Despite similar names, JSON and JavaScript objects have important differences.

Feature JSON JavaScript Object
Key quotes Must use double quotes Optional, usually without quotes
Value types String, Number, Boolean, Null, Object, Array Above + undefined, functions, Symbol
Comments Not supported Supports // and /* */
Trailing comma Not allowed Allowed in modern JS
Type Data format Language-built-in object

Working with JSON in JavaScript

// Object to JSON string
const obj = { name: "John", age: 30 };
const jsonString = JSON.stringify(obj);
console.log(jsonString); // {"name":"John","age":30}

// JSON string to object
const parsed = JSON.parse(jsonString);
console.log(parsed.name); // John

// Formatted output
console.log(JSON.stringify(obj, null, 2));

Common Errors & Solutions

Error 1: Unexpected token

Cause: Special characters in JSON string not escaped
Solution: Use escape characters like \", \\, \n 
// Incorrect
{"text": "He said: "Hello""}

// Correct
{"text": "He said: \"Hello\""}

Error 2: Trailing comma

Cause: Comma after last element
Solution: Remove trailing comma 
// Incorrect
[1, 2, 3, ]

// Correct
[1, 2, 3]

Error 3: Unquoted keys

Cause: JSON keys must be wrapped in double quotes
Solution: Add double quotes to all keys 
// Incorrect
{name: "John"}

// Correct
{"name": "John"}

Error 4: Single quotes

Cause: JSON only supports double quotes
Solution: Replace single quotes with double quotes 
// Incorrect
{'name': 'John'}

// Correct
{"name": "John"}

JSON in API Development

RESTful API Requests

// GET request - fetch data
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data));

// POST request - send JSON data
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'John',
    email: 'john@example.com'
  })
});

JSON Request Body Example

{
  "action": "createUser",
  "data": {
    "username": "johndoe",
    "password": "encrypted_password",
    "profile": {
      "realName": "John Doe",
      "avatar": "https://example.com/avatar.jpg",
      "tags": ["developer", "designer"],
      "settings": {
        "theme": "dark",
        "notifications": true,
        "language": "en-US"
      }
    },
    "createdAt": "2026-03-26T10:30:00Z"
  }
}

Error Response Format

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid email format",
    "field": "email",
    "details": {
      "expected": "Valid email address",
      "received": "invalid-email"
    }
  }
}

JSON Best Practices

Naming Conventions
  • Use camelCase for key names: e.g., firstName, lastName
  • Use is/has/can prefixes for booleans: e.g., isActive, hasPermission
  • Maintain consistency: use uniform naming style throughout the project

1. Use Meaningful Data Structures

// Not recommended: flat structure, poor scalability
{"user_id": 1, "user_name": "John", "user_city": "New York"}

// Recommended: nested structure, clear and extensible
{"user": {"id": 1, "name": "John", "address": {"city": "New York"}}}

2. Handle DateTime Correctly

// Recommended: ISO 8601 format
{"createdAt": "2026-03-26T10:30:00.000Z"}

// Not recommended: custom format
{"createdAt": "2026/3/26 10:30:00"}

3. Use Strings for Enums

// Recommended: clear enum values
{"status": "pending", "role": "admin"}

// Not recommended: numeric magic values
{"status": 0, "role": 1}

4. Pagination Response Format

{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "totalItems": 156,
    "totalPages": 8
  }
}
