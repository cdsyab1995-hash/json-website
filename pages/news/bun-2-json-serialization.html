Bun 2.0 Ships 5x Faster JSON Serialization: A New Benchmark Record

April 21, 2026 | 3-4 min read

Bun 2.0 has arrived, and its headline feature is a completely rewritten JSON serialization engine that clocks in at 5x faster than the previous version — and significantly ahead of Node.js and Deno.

Benchmark Numbers

The Bun team published benchmark results comparing JSON operations across runtimes:

  • Bun 2.0: 1GB JSON serialized in 184ms
  • Bun 1.x: 1GB JSON serialized in 920ms
  • Node.js 24: 1GB JSON serialized in 1,340ms
  • Deno 2.x: 1GB JSON serialized in 1,580ms

What Changed

The new serializer uses SIMD (Single Instruction Multiple Data) instructions available in modern CPUs to process multiple JSON tokens in parallel. The team rewrote the parser's hot paths in Zig, eliminating garbage collection pauses that plagued the previous implementation.

// Bun 2.0 benchmark code
const data = JSON.parse(largePayload);
const serialized = JSON.stringify(data);
// Bun 2.0: ~184ms for 1GB payload
// Node.js 24: ~1,340ms for 1GB payload

Real-World Impact

For applications that process large JSON payloads — think AI agent pipelines, data ingestion pipelines, or high-frequency trading systems — the performance gains translate directly to cost savings and lower latency.
