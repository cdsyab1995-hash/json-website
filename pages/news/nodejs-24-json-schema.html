Node.js 24 Ships Built-in Native JSON Schema Validation

April 19, 2026 | 4-5 min read

Node.js 24 ships a feature that many in the community have requested for years: built-in JSON Schema validation. Powered by Ajv (the fastest JSON Schema validator for JavaScript), this is now available as a native API without any third-party installation.

The New API

// Node.js 24 built-in JSON Schema validation
const { validate, compile } = require('node:schema');

// Compile once, validate many times
const isUser = compile({
  type: 'object',
  properties: {
    id: { type: 'string' },
    email: { type: 'string', format: 'email' },
    role: { enum: ['admin', 'user', 'guest'] }
  },
  required: ['id', 'email']
});

// Validate API payloads
const result = isUser({ id: '123', email: 'test@example.com', role: 'user' });
console.log(result.valid); // true

Why This Matters

Previously, server-side JSON Schema validation required installing packages like Ajv, Zod, or Joi. With Node.js 24, basic validation is available immediately — reducing the dependency footprint for simple use cases.

Draft 2020-12 Support

The built-in validator supports JSON Schema Draft 2020-12, the latest specification, including all new keywords like $dynamicRef, $jsonSchema, and unevaluatedProperties.

For more complex validation scenarios, especially with AI-generated data, dedicated libraries like Zod and Yup still offer more features. But for quick input validation in Express routes or API handlers, Node.js 24's built-in capability is a significant step forward.
