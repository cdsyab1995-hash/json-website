Next.js 16 Introduces Native JSON Streaming and Partial Prerendering

April 19, 2026 | 5-6 min read

Next.js 16 is the framework's biggest release since the App Router, shipping native support for JSON streaming and a reimagined Partial Prerendering model that makes AI-powered pages fast by default.

JSON Streaming in the App Router

Building on the W3C JSON Streaming API now available in all browsers, Next.js 16 adds server-side streaming support for JSON data. This is particularly powerful for pages that call LLM APIs — instead of waiting for the full response, users see content appear incrementally.

// app/api/chat/page.tsx
import { streamJSON } from 'next/streaming';

export default async function ChatPage() {
  return streamJSON({
    stream: llmStream,
    shell: , // Rendered immediately
    chunks: {
      message: async (chunk) => {chunk.text},
      thinking: async (chunk) => {chunk.reasoning},
    }
  });
}

Partial Prerendering Gets Smarter

Next.js 16's Partial Prerendering (PPR) now automatically detects which parts of a page depend on dynamic data (like AI responses) versus static content. Static parts are served from the edge, dynamic parts stream in. The result: sub-100ms LCP even on AI-heavy pages.

JSON as the Transport Layer

The new streaming model uses JSON as the primary data transport between server components and client components. This makes Next.js's internal data model more explicit and easier to debug — every component boundary passes typed JSON.
