Browser DevTools Now Built-in JSON Schema Validation and Live Error Highlighting

April 21, 2026 | 3-4 min read

Chrome 122 and Firefox 120 shipped a feature that developers have wanted for years: native JSON Schema validation directly in DevTools. Now you can paste a JSON Schema document and see real-time validation errors highlighted as you type.

How It Works

Open the Sources panel in Chrome DevTools (or the Debugger in Firefox), navigate to the JSON Schema tab, and paste your schema. Then paste any JSON document you want to validate — errors appear inline with the exact path to the problematic field.

{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer", "minimum": 0 }
  },
  "required": ["name"]
}

What Changed in Chrome 122

Chrome's implementation uses the native ajv library under the hood, supporting JSON Schema Draft 2020-12. The validation panel integrates with the Network panel, so you can right-click any API response and choose "Validate against schema" — no copy-pasting required.

Firefox's Approach

Firefox took a slightly different angle, embedding schema validation into the JSON viewer itself. Toggle "Schema validation" in the viewer toolbar, paste your schema URL, and the viewer highlights violations directly in the response tree.

Impact on Developer Workflow

This native integration means fewer round-trips to external validators. For teams using JSON Schema for API contracts, schema validation in DevTools closes the gap between writing a schema and testing it against real data.

For more advanced schema editing, AI JSON Formatter still offers syntax highlighting, error explanations, and template generation — complementing what DevTools now provides out of the box.
