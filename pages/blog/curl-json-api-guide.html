Mastering curl with JSON APIs: The Complete Command-Line Guide

Published: 2026-04-18 · 6-8 min read

Every developer needs to debug, test, or script against JSON APIs. curl is the universal tool — pre-installed on macOS and Linux, available everywhere. This guide covers everything from basic GET requests to handling authentication, pagination, and pretty-printing responses.

Why curl?

GUIs like Postman are great for exploration, but curl has permanent advantages:

  • Zero setup: Comes pre-installed on nearly every system
  • Scriptable: Embed in shell scripts, CI pipelines, Makefiles
  • Reproducible: Share exact API calls as text in issues and documentation
  • Lightweight: No GUI overhead, works over SSH, inside containers

Basic GET Request

curl https://jsonplaceholder.typicode.com/users/1

Returns raw JSON on one line. Not very readable — here is how to fix that.

Pretty-Printing JSON Responses

# Option 1: Pipe through jq (most powerful, install with brew/apt)
curl -s https://jsonplaceholder.typicode.com/users/1 | jq

# Option 2: Python stdlib (no install needed)
curl -s https://jsonplaceholder.typicode.com/users/1 | python -m json.tool

# Option 3: curl built-in (limited formatting)
curl https://jsonplaceholder.typicode.com/users/1 --indent 2

jq is the gold standard — it transforms, filters, and formats JSON in powerful ways.

Common curl Flags for JSON APIs

-s, --silent       # Suppress progress meter (clean output)
-S, --show-error   # Show errors even with -s
-i, --include      # Include response headers in output
-I, --head         # HEAD request (headers only)
-w, --write-out    # Print custom format after completion
-o, --output FILE  # Write output to file
-X, --request METHOD  # Specify HTTP method (GET/POST/PUT/PATCH/DELETE)
-H, --header       # Add request header
-d, --data         # Request body data
-A, --user-agent   # Set User-Agent header

GET with Query Parameters

# Filter with query params
curl "https://jsonplaceholder.typicode.com/posts?userId=1"

# Multiple params
curl "https://api.example.com/search?q=json&limit=10&offset=0"

POST: Sending JSON Data

curl -X POST https://jsonplaceholder.typicode.com/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "My Post", "body": "Content here", "userId": 1}'

# Pretty print response
curl -s -X POST https://jsonplaceholder.typicode.com/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "My Post", "body": "Content here", "userId": 1}' | jq

PUT: Full Resource Replacement

curl -X PUT https://jsonplaceholder.typicode.com/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"id": 1, "title": "Updated Title", "body": "Updated body", "userId": 1}'

PATCH: Partial Updates

curl -X PATCH https://jsonplaceholder.typicode.com/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"title": "Only updating the title"}'

DELETE

curl -X DELETE https://jsonplaceholder.typicode.com/posts/1

Authentication: Bearer Token

# Bearer token (most common)
curl -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  https://api.example.com/user/profile

# API Key as header
curl -H "X-API-Key: YOUR_API_KEY" \
  https://api.example.com/data

# Basic Auth
curl -u "username:password" \
  https://api.example.com/private

Saving Responses to File

# Download JSON response
curl -s https://api.example.com/data -o response.json

# Download binary (images, PDFs)
curl -L -o output.pdf https://api.example.com/reports/2024.pdf

Inspecting HTTP Headers

# Show request and response headers
curl -i https://jsonplaceholder.typicode.com/users/1

# Response headers only
curl -I https://jsonplaceholder.typicode.com/users/1

# Check rate limits
curl -I https://api.github.com/users/octocat
# HTTP/2 200
# x-ratelimit-remaining: 59

Following Redirects

# Follow redirects (like a browser)
curl -L https://short.url/abc123

Pagination: Fetching All Pages

# Fetch all users handling pagination
page=1
while true; do
  response=$(curl -s "https://jsonplaceholder.typicode.com/posts?_page=$page&_limit=10")
  if [ -z "$response" ] || [ "$response" = "[]" ]; then break; fi
  echo "$response"
  page=$((page + 1))
done | jq -s '. | add'

Checking Response Time

curl -w "DNS: %{time_namelookup}s | TCP: %{time_connect}s | TTFB: %{time_starttransfer}s | Total: %{time_total}s
" \
  -o /dev/null -s https://jsonplaceholder.typicode.com/users

Debugging: Verbose Output

# Show everything: requests, headers, SSL handshake
curl -v https://api.example.com/data 2>&1

# Maximum verbosity (including certificate details)
curl --trace-ascii - https://api.example.com/data

Useful jq One-Liners

# Pretty-print any JSON
cat data.json | jq .

# Extract a specific field
curl -s https://api.github.com/users/octocat | jq '. followers'

# Transform structure
curl -s https://api.example.com/users | jq '.[] | {name, email}'

# Count items
curl -s https://api.example.com/users | jq 'length'

# Filter and format
curl -s https://api.github.com/users/octocat/repos | \
  jq '.[] | select(.fork == false) | .name'

# Check API health
curl -s https://api.example.com/health | jq '.status'

Using Environment Variables for Secrets

Never hardcode tokens in scripts:

# Set token once
export API_TOKEN="sk-..."

# Use in curl commands
curl -H "Authorization: Bearer $API_TOKEN" \
  https://api.example.com/user

Key Takeaways

  • -H "Content-Type: application/json" is required for JSON POST/PUT/PATCH bodies
  • Use jq for powerful JSON manipulation — it is worth the install
  • -s keeps output clean by hiding the progress meter
  • -w lets you benchmark response times
  • Store tokens in environment variables, never in scripts
  • Share exact curl commands in bug reports — it is the universal language
