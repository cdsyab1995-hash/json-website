什么是JSON？

JSON (JavaScript Object Notation) 是一种轻量级的数据交换格式，由Douglas Crockford在2001年推广。它基于JavaScript的一个子集，但独立于编程语言，几乎所有现代编程语言都支持JSON。

为什么学习JSON？
  • API开发：95%以上的Web API使用JSON传输数据
  • 配置文件：package.json、tsconfig.json等
  • 数据存储：NoSQL数据库、配置文件
  • 日志系统：结构化日志记录

JSON的特点

  • 简洁易读：相比XML更紧凑，结构清晰
  • 易于解析：所有主流编程语言都有内置支持
  • 跨平台：语言无关，任意系统间可交换
  • 类型安全：支持多种数据类型

JSON基础语法

JSON有严格的语法规则，违反了任何一条都会导致解析失败。

基本结构

{
  "name": "张三",
  "age": 28,
  "isStudent": false,
  "skills": ["JavaScript", "Python", "SQL"],
  "address": {
    "city": "北京",
    "district": "朝阳区"
  }
}

核心规则

规则 正确示例 错误示例
键必须用双引号 "name" name 或 'name'
字符串必须双引号 "Hello" 'Hello'
不能有尾随逗号 [1, 2, 3] [1, 2, 3, ]
不能有注释 （无注释） // 注释
不能有undefined 使用 null "value": undefined

JSON数据类型

JSON支持6种基本数据类型。

1. 字符串 (String)

{
  "message": "Hello World",
  "unicode": "中文和emoji😊",
  "escape": "换行符\\n和制表符\\t"
}

2. 数字 (Number)

{
  "integer": 42,
  "float": 3.14159,
  "negative": -17,
  "scientific": 1.5e10,
  "hex": 0xFF
}

3. 布尔值 (Boolean)

{
  "isActive": true,
  "isDeleted": false
}

4. 空值 (Null)

{
  "middleName": null,
  "deletedAt": null
}

5. 对象 (Object)

{
  "user": {
    "id": 1,
    "name": "李四",
    "email": "li@example.com"
  }
}

6. 数组 (Array)

{
  "colors": ["red", "green", "blue"],
  "numbers": [1, 2, 3, 4, 5],
  "mixed": [1, "two", true, null],
  "objects": [
    {"id": 1, "name": "A"},
    {"id": 2, "name": "B"}
  ]
}

JSON与JavaScript对象

虽然名字相似，但JSON和JavaScript对象有重要区别。

特性 JSON JavaScript对象
键引号 必须双引号 可选，通常无引号
值类型 字符串、数值、布尔、null、对象、数组 上述 + undefined、函数、Symbol
注释 不支持 支持 // 和 /* */
trailing comma 不允许 现代JS允许
类型 数据格式 语言内置对象

JavaScript中处理JSON

// 对象转JSON字符串
const obj = { name: "张三", age: 30 };
const jsonString = JSON.stringify(obj);
console.log(jsonString); // {"name":"张三","age":30}

// JSON字符串转对象
const parsed = JSON.parse(jsonString);
console.log(parsed.name); // 张三

// 带格式化的输出
console.log(JSON.stringify(obj, null, 2));

常见错误与解决方案

错误1: Unexpected token

原因：JSON字符串中包含了特殊字符未转义
解决：使用转义字符，如 \"、\\、\n 
// 错误
{"text": "他说: "你好""}

// 正确
{"text": "他说: \"你好\""}

错误2: Trailing comma

原因：最后一个元素后面有逗号
解决：移除尾随逗号 
// 错误
[1, 2, 3, ]

// 正确
[1, 2, 3]

错误3: Unquoted keys

原因：JSON的键必须用双引号包裹
解决：给所有键加上双引号 
// 错误
{name: "张三"}

// 正确
{"name": "张三"}

错误4: Single quotes

原因：JSON只支持双引号
解决：将单引号替换为双引号 
// 错误
{'name': '张三'}

// 正确
{"name": "张三"}

API开发中的JSON

RESTful API 请求

// GET 请求 - 获取数据
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data));

// POST 请求 - 发送JSON数据
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: '张三',
    email: 'zhang@example.com'
  })
});

JSON请求体示例

{
  "action": "createUser",
  "data": {
    "username": "zhangsan",
    "password": "encrypted_password",
    "profile": {
      "realName": "张三",
      "avatar": "https://example.com/avatar.jpg",
      "tags": ["开发者", "设计师"],
      "settings": {
        "theme": "dark",
        "notifications": true,
        "language": "zh-CN"
      }
    },
    "createdAt": "2026-03-26T10:30:00Z"
  }
}

错误响应格式

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "邮箱格式不正确",
    "field": "email",
    "details": {
      "expected": "有效的邮箱地址",
      "received": "invalid-email"
    }
  }
}

JSON最佳实践

命名规范
  • 键名使用 camelCase：如 firstName、lastName
  • 布尔值用 is/has/can 前缀：如 isActive、hasPermission
  • 保持一致性：整个项目使用统一的命名风格

1. 使用有意义的数据结构

// 不推荐：扁平结构，扩展性差
{"user_id": 1, "user_name": "张三", "user_city": "北京"}

// 推荐：嵌套结构，清晰易扩展
{"user": {"id": 1, "name": "张三", "address": {"city": "北京"}}}

2. 正确处理日期时间

// 推荐：ISO 8601格式
{"createdAt": "2026-03-26T10:30:00.000Z"}

// 不推荐：自定义格式
{"createdAt": "2026/3/26 10:30:00"}

3. 枚举值使用字符串

// 推荐：清晰的枚举值
{"status": "pending", "role": "admin"}

// 不推荐：数字魔法值
{"status": 0, "role": 1}

4. 分页响应格式

{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "totalItems": 156,
    "totalPages": 8
  }
}
